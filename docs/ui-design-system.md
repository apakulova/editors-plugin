# UI-дизайн-система «Чистовика»

Этот документ фиксирует общую визуальную систему интерфейса плагина. Все новые экраны, состояния и сообщения должны опираться на нее: не добавляем кастомные кегли, веса, отступы, цвета и состояния, если это не нужно по явной причине, не согласовано отдельно и не записано здесь.

Markdown в Codex не всегда показывает локальные картинки и HTML-стили внутри таблиц. Поэтому этот файл хранит надежную текстовую версию дизайн-системы, а визуальная версия с настоящими цветами и кеглями лежит рядом: `docs/ui-design-system-preview.html`.

## Базовые правила

- Основной источник правды для UI-стилей — `src/ui.html`.
- Тексты вкладок хранятся в `src/ui-content.js`, но визуальные стили задаются в `src/ui.html`.
- Для UI-доработок сначала ищем подходящий существующий стиль из таблицы ниже.
- Если существующего стиля не хватает, новый стиль нужно согласовать и добавить в этот документ.
- Правила типографики текста в макете не связаны с UI-стилями плагина. При работе над интерфейсом не менять `cleanTypography`, порядок правил, regex-замены и тестовые ожидания.

## Шрифт

Во всем интерфейсе используется единая системная цепочка:

```css
font-family: Manrope, Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

## Текстовая шкала

Общие правила:

- весь жирный текст использует `font-weight: 700`;
- обычный текст использует `font-weight: 400`;
- кегль `12px` всегда идет с `line-height: 16px`;
- мелкий кегль `11px` идет с `line-height: 15px`;
- заголовки `16px` идут с `line-height: 20px`;
- основной текст — `#141414`, вторичный — `#757575`.

## Ссылки

Ссылка в UI опирается на стиль ссылки на канал во вкладке `О плагине`:

```css
color: #065C5B;
font-weight: 700;
text-decoration: none;
```

Ссылка наследует кегль и line-height от родительского текста. Например, в авторской плашке она наследует `11px / 15px`, а в подзаголовке окна ошибок — `12px / 16px`.

## Маркеры и декоративные символы

- Маркер пункта в правилах и подсказках — `✦`, цвет `#065C5B`, `font-size: 10px`, `line-height: 16px`.
- Звездочка режима `Для разработки` внутри текста — `*`, цвет `#FF4053`, `font-weight: 700`.
- Большие декоративные звездочки на вкладке `О плагине` — часть существующего экрана; новые декоративные символы не добавляем без отдельного согласования.

## Отступы

Основная сетка:

- контент вкладок: `padding: 24px 16px 12px`;
- горизонтальный базовый отступ от края окна: `16px`;
- заголовок + контент группы: `gap: 14px`;
- список режимов: `gap: 12px`;
- список правил: `gap: 8px`;
- пункты внутри раскрытого правила: `gap: 6px`;
- подзаголовок в правой части окна ошибок + следующий текст или список: `gap: 6px`, как у пунктов внутри раскрытого правила;
- контент строки ошибки: `padding: 7px 16px`, чтобы текст начинался там же, где заголовок окна.

## Компоненты

Основная кнопка:

- `background: #065C5B`;
- hover: `#054A49`;
- текст: `14px / 18px`, `font-weight: 500`, белый;
- `height` или `min-height`: `40px`;
- `border-radius: 8px`;
- `padding: 0 14px`.

Активная строка в списке ошибок:

- `background: #F0F4F3`;
- без рамки;
- без радиуса;
- заливка идет на всю ширину строки.

Аккордеон правил:

- заголовок: `12px / 16px`, `font-weight: 700`;
- раскрытый фон: `#F0F4F3`;
- пункты: `12px / 16px`, `font-weight: 400`, цвет `#757575`;
- примеры: `11px / 15px`, `font-weight: 700`, цвет `#141414`.

## UI-отчет об ошибках

Окно ошибок — часть общей дизайн-системы, а не отдельная стилистика.

Основные тексты:

- общая ошибка запуска: `Не удалось запустить типограф`;
- ошибки конкретных текстовых слоев: `Не удалось обработать N слой/слоя/слоёв`;
- подзаголовок: Нажмите на строчку, чтобы посмотреть подсказки. Если исправить не выходит, пришлите Ане Акуловой в ТГ ([@akanna](https://t.me/akanna)) скриншот и ссылку на файл — будем разбираться ♡

Общий заголовок отчета:

- стиль: роль `Заголовок экрана` из таблицы;
- `16px / 20px`, `font-weight: 700`, цвет `#141414`;
- отступ блока заголовка: `padding: 16px 16px 14px`;
- `gap: 6px`;
- снизу граница `1px solid #DBDBD9`.

Подзаголовок отчета:

- стиль: роль `Основной поясняющий текст`;
- `12px / 16px`, `font-weight: 400`, цвет `#757575`.

Левая колонка:

- ширина в preview: `244px`;
- фон белый, без серой заливки всей колонки;
- строка ошибки занимает всю ширину;
- строка ошибки: `padding: 7px 16px`, `min-height: 58px`, `gap: 3px`;
- левый отступ строки — `16px`, как у заголовка окна;
- активная строка: `background: #F0F4F3`;
- без border и без border-radius вокруг активной строки.

Текст строки:

- основной идентификатор — `textPreview`;
- стиль `textPreview`: роль `Основной текст строки`, `12px / 16px`, `font-weight: 400`, цвет `#141414`;
- максимум 2 строки, без жирности;
- название слоя или путь — роль `Мелкая вторичная подпись`, `11px / 15px`, `font-weight: 400`, цвет `#757575`;
- тип ошибки в левой строке не показывать.

Правая колонка:

- контейнер: `padding: 12px 16px 8px`;
- `gap: 8px`;
- `align-content: start`;
- заголовок ошибки: роль `Заголовок экрана`;
- код ошибки: роль `Мелкая вторичная подпись`;
- подзаголовки `Что случилось` и `Что можно попробовать`: роль `Подзаголовок / заголовок группы`;
- текст в блоке `Что случилось`: роль `Основной поясняющий текст`;
- пункты в `Что можно попробовать`: роль `Основной поясняющий текст`, маркеры как в правилах.
- расстояние от подзаголовка до текста или списка: `gap: 6px`.

Кнопка `Показать в макете`:

- стиль основной кнопки;
- ширина по содержимому, не на всю колонку;
- для общей ошибки кнопку `Закрыть` не показывать, потому что в окне уже есть крестик.

## Как добавлять новые UI-стили

Если нужен новый стиль:

1. Сначала проверить, нельзя ли использовать роль из таблицы выше.
2. Если нельзя — согласовать отличие отдельно.
3. Записать новый стиль в этот документ.
4. Только после этого использовать его в `src/ui.html` или preview-файлах.
