# Журнал коммитов

Этот файл хранит рабочую память проекта между чатами. После каждого нового локального коммита нужно добавлять сюда запись на русском языке: код коммита, что изменилось, зачем это было сделано, какие файлы затронуты, какие проверки прошли и что важно не потерять в следующем чате.

README должен оставаться коротким: полный журнал ведется здесь, а в README остается только ссылка на этот файл.

- `e3f0e66` — `Починить отправку аналитики`.
  Исправлена отправка событий в PostHog: из Capture payload убрано кастомное поле `uuid`, потому что PostHog принимает только отсутствующий `uuid` или настоящий UUID-формат, а внутренние id вида `evt_...` приводили к `400 Bad Request`. Внутренний `eventId` оставлен только как id записи локальной очереди. Чтение очереди `analyticsEventQueue` теперь нормализует legacy-payload перед отправкой, чтобы старые события с плохим `uuid` не блокировали новые события.
  Аналитика больше не задерживает запуск из окна настроек и закрытие UI: ожидание отправки событий убрано из UI-сценариев. Для быстрого запуска оставлен короткий grace period `500 ms` перед закрытием плагина. При обработке выделенного текста, фрейма, группы или другого контейнера плагин больше не вызывает `figma.currentPage.loadAsync()` заранее; загрузка текущей страницы осталась только для сценария без выделения, когда нужно обработать всю страницу.
  README и `docs/analytics-plan.md` обновлены: зафиксированы формат PostHog payload без кастомного `uuid`, локальная очередь аналитики и условная загрузка страницы. Добавлен регрессионный тест, что `createAnalyticsEventPayload` не содержит `uuid`; `dist/code.js` пересобран.
  Затронуты `src/code.ts`, `scripts/test-clean-typography.js`, сгенерированный `dist/code.js`, README и `docs/analytics-plan.md`. `cleanTypography`, порядок типографических правил, тексты уведомлений, UI-тексты, manifest, Telegram/Vercel-код не менялись.
  Проверки: `npm test`, `git diff --check`, ручная диагностическая отправка тестового события `is_test_event: true` в PostHog вернула `200 OK`. В Figma после обновления нужно проверить быстрый запуск, запуск из настроек по выделенному фрейму, запуск без выделения по странице, появление уведомлений успеха/ошибки и появление реальных событий в PostHog/Telegram-отчете после деплоя.
- `79377a4` — `Исправить сокращения и сохранение style id`.
  Уточнены правила составных сокращений: кириллические сокращения `т. е.`, `т. к.`, `т. д.`, `т. п.`, `н. в.`, `и т. д.`, `и т. п.` и `в т. ч.` теперь требуют явный разделитель между частями — точку или пробел. Поэтому слитные слова и аббревиатуры вроде `Те`, `ТК` и `НВ` больше не раскрываются в сокращения. Если сокращение распознано, регистр кириллических букв сохраняется: `Т е` -> `Т. е.`, `т к` -> `т. к.`. Постскриптум оставлен отдельным эталонным правилом: `PS` и `PPS` приводятся к `P. S.` и `P. P. S.`.
  Дополнительно сохранение оформления стало надежнее для style id: `captureTextStyles` теперь добирает `textStyleId` и `fillStyleId` из range/node fallback и посимвольной проверки, а восстановление применяет `textStyleId` перед `fillStyleId`. Это нужно, чтобы после изменения `textNode.characters` не терять подключенные текстовые и цветовые стили.
  Затронуты `src/code.ts`, сгенерированный `dist/code.js` и `scripts/test-clean-typography.js`. README, UI-тексты, уведомления, настройки запуска, manifest, аналитика, Telegram/Vercel и правила вне блока сокращений не менялись.
  Проверки: `npm test`, `git diff --check`, ручная проверка фактических результатов `ТК`, `НВ`, `т к`, `н в`, `PS`, `PPS`. В Figma после обновления нужно проверить текстовые слои с `Те`, `т.е.`, `т е`, `ТК`, `НВ`, `PS`, `PPS`, а также слой с подключенными text style и fill style после запуска типографа.

- `5950441` — `Уточнить правила звездочек и формул`.
  Изменено правило кавычек: многоточие больше не переносится через закрывающую кавычку и остается там, где стояло до обработки, внутри кавычек или после них.

Она доступна только в режиме `Для разработки`: безопасные одиночные пользовательские `*` временно считаются пробельными кандидатами, после чего типограф решает, заменить их обычным пробелом или служебным красным маркером неразрывного пробела. Серии звездочек, маски карт/паролей, латинские коды, email-like токены и небезопасные случаи остаются пользовательскими.
  Уточнены безопасные сценарии для существующих `*`: пробелы между кириллическими словами, внутри скобок, рядом с тире и между цифрой и кириллическим словом. Защищены кейсы `карта ****4444`, `карта****4444`, `SALE*2026` и `SALE*2*026`.
  Математика с умножением теперь распознает компактную формулу перед запятой: `2*2,` превращается в `2 × 2,`, а в режиме разработки — в `2*×*2,`.
