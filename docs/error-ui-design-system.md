# Дизайн-система UI-отчета об ошибках

Документ фиксирует визуальные правила для будущего окна отчета об ошибках обработки. Его задача — не дать новому экрану уехать в отдельную стилистику. Отчет должен выглядеть как часть текущего UI «Чистовика».

## Основной принцип

UI-отчет об ошибках должен использовать те же цвета, кегли, веса, радиусы и плотность, что текущий интерфейс плагина в `src/ui.html`.

Нельзя:

- вводить отдельную красную палитру для ошибок;
- использовать голубую подсветку строк;
- делать крупные dashboard-заголовки;
- использовать карточки внутри карточек;
- добавлять рамку вокруг активной строки списка;
- растягивать кнопку на всю ширину, если команда короткая;
- делать правую часть просторнее, чем текущие вкладки `Правила` и `О плагине`.

## Цвета

Использовать текущие CSS-токены плагина:

- основной текст: `#141414`;
- вторичный текст: `#757575`;
- граница: `#DBDBD9`;
- фон окна: `#FFFFFF`;
- вторичный фон: `#F7F7F5`;
- акцент: `#065C5B`;
- hover акцента: `#054A49`;
- мягкая акцентная заливка: `#F0F4F3`.

Для активной строки в списке ошибок использовать только мягкую акцентную заливку:

```css
background: #F0F4F3;
```

Без рамки, без радиуса и без внешних отступов вокруг заливки. Заливка должна идти на всю ширину строки, как выделение результата поиска в Figma.

Красный `#FF4053` остается только для служебных звездочек режима `Для разработки`; для UI ошибок его не использовать.

## Окно

Preview-окно для согласования:

- ширина: `640px`;
- высота содержимого отчета: около `480px`;
- верхняя панель окна: `52px`;
- внешний радиус окна в preview: `14px`, как в существующем локальном предпросмотре;
- внутренние элементы используют радиус `8px` или `0`, в зависимости от паттерна.

В реальном Figma-плагине размер окна можно уточнить после интеграции, но плотность интерфейса должна остаться такой же.

## Общий заголовок отчета

Главный заголовок над таблицей:

- класс в preview: `.report-title`;
- кегль: `16px`;
- line-height: `20px`;
- font-weight: `800`;
- цвет: `#141414`;
- margin: `0`.

Тексты:

- для общей ошибки запуска: `Не удалось запустить типограф`;
- для ошибок конкретных текстовых слоев: `Не удалось обработать N слой/слоя/слоёв`;
- слово `слой` склоняется по числу.

Подзаголовок:

- класс в preview: `.report-text`;
- кегль: `12px`;
- line-height: `17px`;
- цвет: `#757575`;
- margin: `0`.

Текст:

```text
Нажмите на строчку, чтобы посмотреть подсказки. Если исправить не выходит, пришлите Ане Акуловой в ТГ (@akanna) скриншот и ссылку на файл — будем разбираться ♡
```

Отступы блока заголовка:

- `padding: 16px 16px 14px`;
- `gap: 6px`;
- снизу граница `1px solid #DBDBD9`.

## Левая колонка

Ширина левой колонки в preview:

```css
grid-template-columns: 244px 1fr;
```

Фон левой колонки: белый. Не использовать серую заливку всей колонки.

Строка ошибки:

- занимает всю ширину колонки;
- `padding: 7px 10px`;
- `min-height: 58px`;
- `gap: 3px`;
- без border;
- без border-radius;
- фон активной строки: `#F0F4F3`;
- фон неактивной строки: transparent.

Основной идентификатор строки — `textPreview`:

- кегль: `12px`;
- line-height: `16px`;
- font-weight: `400`;
- цвет: `#141414`;
- максимум 2 строки;
- без жирности.

Название слоя вторично:

- кегль: `11px`;
- line-height: `14px`;
- цвет: `#757575`;
- показывать мелко серым или не показывать вовсе.

Тип ошибки в левой строке не показывать. Он отображается только в правой части.

## Правая колонка

Правая часть должна быть компактной, как содержимое аккордеонов на вкладке `Правила`.

Контейнер:

- `padding: 12px 16px 8px`;
- `gap: 8px`;
- `align-content: start`.

Заголовок ошибки:

- класс в preview: `.detail-title`;
- кегль: `13px`;
- line-height: `17px`;
- font-weight: `800`;
- цвет: `#141414`;
- margin: `0`.

Код ошибки:

- класс в preview: `.code`;
- кегль: `10px`;
- line-height: `13px`;
- цвет: `#757575`;
- должен стоять сразу под заголовком, без визуальной дырки.

Подзаголовки секций:

- `Что случилось`;
- `Что можно попробовать`;
- кегль: `12px`;
- line-height: `16px`;
- font-weight: `800`;
- цвет: `#141414`.

Отступ между подзаголовком секции и текстом:

```css
gap: 2px;
```

Текст секций:

- кегль: `12px`;
- line-height: `16px`;
- цвет: `#757575`;
- margin: `0`.

Список подсказок:

- кегль: `12px`;
- line-height: `16px`;
- цвет: `#757575`;
- `gap: 1px`;
- маркер: акцентная звездочка `✦` цвета `#065C5B`, как в правилах и вкладке `О плагине`.

Бейджи `Общая ошибка` и `Текстовый слой` не использовать.

## Кнопки

Кнопка должна повторять основную кнопку плагина:

- фон: `#065C5B`;
- hover: `#054A49`;
- текст: белый;
- кегль: `14px`;
- font-weight: `500`;
- line-height: `18px`;
- min-height: `40px`;
- border-radius: `8px`;
- padding: `0 14px`;
- ширина по содержимому, а не на всю колонку.

Для общей ошибки кнопку `Закрыть` не показывать: в окне уже есть крестик.

Не добавлять рядом с кнопкой поясняющий текст. Не добавлять разделительную линию над кнопкой.

## Preview

Статический предпросмотр живет здесь:

```text
preview/error-report-preview.html
```

Он нужен для согласования внешнего вида до интеграции в Figma. Preview должен оставаться близким к реальному UI и использовать правила из этого документа.

Скриншот preview можно обновить через headless Chrome, но PNG не является обязательной частью коммита.
